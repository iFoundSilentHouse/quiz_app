# ✅ Шаг 5 — Редактирование тестов, UI и конфигурация окружения

## Что было сделано

### 1. Бэкенд (Nest.js API)
- **Настройка CORS (`main.ts`)**: 
  - [cite_start]Захардкоженные порты заменены на использование переменной окружения `FRONTEND_PORT`[cite: 2].
  - [cite_start]Явно прописаны разрешенные HTTP-методы: `GET, HEAD, PUT, PATCH, POST, DELETE` для корректной работы Preflight-запросов (OPTIONS) из браузера[cite: 2].
- **Новый контроллер вопросов (`question.controller.ts`)**: 
  - [cite_start]Создан выделенный `QuestionController`, который взял на себя обработку точечного обновления (`PUT`) и удаления (`DELETE`) вопросов по их ID[cite: 5]. Это позволило разгрузить основной `QuizController`.
- **Связи TypeORM (`quiz.service.ts`)**:
  - В метод `findOne` добавлено свойство `relations: ['questions']`, чтобы при запросе теста сервер возвращал массив связанных с ним вопросов.

### 2. Фронтенд (Next.js Web)
- **Обновление системы стилей**: 
  - Проект переведен на Tailwind CSS v4. [cite_start]Добавлены зависимости `@tailwindcss/postcss` (^4.2.2), `postcss` (^8.5.8) и `tailwindcss` (^4.2.2)[cite: 6]. 
  - [cite_start]Зафиксированы новые транзитивные зависимости в `pnpm-lock.yaml`, включая сборщик `lightningcss` для быстрой компиляции стилей[cite: 14, 15, 16, 17, 18].
- **Настройка абсолютных импортов (`tsconfig.json`)**: 
  - [cite_start]Добавлены параметры `baseUrl: "."` и `paths: { "@/*": ["./src/*"] }`[cite: 7]. Это позволяет использовать чистые импорты вида `import { api } from '@/lib/api'`.
- **Поддержка асинхронного API Next.js 15**: 
  - Страница `/quizzes/[id]/edit/page.tsx` обновлена: добавлен `await params` для корректного извлечения ID теста из URL.

### 3. Компоненты UI (React)
- **Строгая типизация состояний**: Введен интерфейс `QuestionState` с опциональными полями (`id?`, `imageUrl?`, `correctAnswer?`), что решило проблемы с `undefined` и типизацией в методах `.filter()` и `.map()`.
- **Универсальная форма `QuizForm`**: Единый компонент теперь обрабатывает как создание (POST), так и редактирование (PUT) тестов, получая данные через пропс `initialData`.
- **Безопасные итерации**: Внедрен цикл `for...of` с `Array.entries()` для надежной асинхронной отправки вопросов на сервер.

📖 Руководство по использованию UI (Интерфейс Учителя)
1. Создание нового теста

    Перейдите на главную страницу учителя: /dashboard.

    Нажмите кнопку «Создать тест».

    Заполните основные данные: Название (обязательно) и Описание.

    Добавление вопросов:

        Кликните на пунктирную область, чтобы загрузить изображение с вашего компьютера. Картинка автоматически отправится на сервер, и появится её превью.

        Введите Правильный ответ (слово на английском, например: bought).

        Нажмите «+ Добавить вопрос», чтобы создать дополнительные карточки.

    Нажмите «Сохранить тест». Вы будете перенаправлены обратно на дашборд.

2. Редактирование существующего теста

    На странице /dashboard найдите нужный тест в списке и нажмите «Редактировать».

    Форма автоматически загрузит текущее название, описание и все ранее созданные вопросы (благодаря initialData).

    Доступные действия:

        Изменить текст: Отредактируйте название теста или правильные ответы в существующих карточках.

        Заменить картинку: Нажмите на текущее изображение, чтобы выбрать новый файл.

        Удалить вопрос: Нажмите кнопку «Удалить» в правом верхнем углу карточки. Если вопрос уже есть в базе данных, система запросит подтверждение (Confirm) и сразу удалит его через API.

        Добавить новые: Нажмите «+ Добавить вопрос».

    Нажмите «Сохранить изменения». Система автоматически обновит тест (PUT), обновит старые вопросы (PUT) и создаст новые (POST).
